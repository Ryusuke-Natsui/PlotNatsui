# 設計の道のり（ロードマップ）

## 0. 目標整理
このアプリの目的は、ラマン・PL などのスペクトルデータについて  
**「読み込む → 整える → 比べる → 図に仕上げる」** を 1 つのブラウザアプリで行えるようにすることです。

最初から全部入りにすると設計が重くなるため、以下の 4 段階で進めるのがおすすめです。

---

## 1. Phase 1: 最小実用版（MVP）
### 目的
まずは **日常的に使える最低限のプロッタ** を作る。

### 入れる機能
- txt / csv / tsv 読み込み
- 複数スペクトル重ね描き
- 表示 / 非表示切り替え
- 縦方向オフセット
- x軸 / y軸ラベル変更
- 簡易ピーク検出
- 選択ピークで正規化
- PNG 書き出し
- プロジェクト設定保存

### 技術方針
- ビルド不要
- GitHub Pages でそのまま公開
- Plotly を描画エンジンとして利用
- 状態管理を分離して、後で機能追加しやすくする

### 完了条件
- 自分の測定ファイルをブラウザに入れて図が作れる
- 複数スペクトル比較ができる
- 最低限の図の出力ができる

---

## 2. Phase 2: 解析強化
### 目的
ピーク比較や定量に耐えるようにする。

### 入れる機能
- スムージング
- ベースライン補正
- cosmic ray 除去
- ピーク検出条件調整
- 検出ピークの手動修正
- 任意範囲積分
- 差スペクトル / 比スペクトル
- 複数スペクトルの一括正規化

### 設計ポイント
- 前処理をパイプライン化する
- 「元データ」と「処理後データ」を分離する
- 各処理を ON/OFF 可能にする

### 完了条件
- 同じデータに何度でも同じ処理を再現できる
- どの処理を適用したか記録できる

---

## 3. Phase 3: フィットと図仕上げ
### 目的
論文図に近いレベルまで持っていく。

### 入れる機能
- Gaussian / Lorentzian / Voigt フィット
- 複数ピーク同時フィット
- 背景込みフィット
- フィット残差表示
- inset 図
- 凡例・フォント・余白の細かい調整
- 縦横比の指定
- SVG / PDF 書き出し
- ピークラベルテンプレート

### 設計ポイント
- 解析画面と図仕上げ画面を分ける
- フィット結果を表形式で保存する

---

## 4. Phase 4: 高度化
### 目的
研究グループ内で使い回せるツールにする。

### 入れる機能
- バッチ処理
- ピークライブラリ
- マッピング / ヒートマップ
- 温度・時間依存データの waterfall 表示
- 自動レポート出力
- 共有テンプレート
- 材料系ごとのプリセット

### 完了条件
- 複数人が同じルールで使える
- 解析条件の再現性が高い
- 一括処理に耐える

---

## UI 設計の考え方
おすすめ構成は以下です。

### 左カラム
- 読み込んだスペクトル一覧
- 表示 ON/OFF
- 色 / オフセット / 名前
- 選択状態

### 中央
- メインプロット

### 右カラム
- 前処理
- ピーク検出
- 正規化
- 図のデザイン設定
- エクスポート

### 下部
- ピーク表
- 処理ログ
- フィット結果

---

## データモデルの考え方
スペクトル 1 本あたり以下のような情報を持つと拡張しやすいです。

- id
- name
- sourceFileName
- xRaw
- yRaw
- xProcessed
- yProcessed
- visible
- color
- lineWidth
- offset
- normalization
- detectedPeaks
- labels
- metadata
